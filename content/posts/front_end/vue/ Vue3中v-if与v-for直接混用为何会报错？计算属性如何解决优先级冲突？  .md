### v-if与v-for的基础作用

在Vue3的模板语法中，`v-if`和`v-for`是两个高频使用的指令：

- **v-if**：条件渲染指令，根据表达式的真假决定是否渲染元素（或组件），会直接销毁/重建DOM元素。
- **v-for**：列表渲染指令，基于数组或对象循环生成多个元素，需要通过`:key`指定唯一标识以优化渲染。

### 核心规则：v-if优先级高于v-for

在Vue3中，**v-if的优先级高于v-for**——当两个指令同时作用于**同一个元素**时，`v-if`会先执行，而`v-for`尚未开始循环。这会导致一个关键问题：
**`v-if`的表达式无法访问`v-for`的循环变量**。

#### 错误示例：直接共存导致的问题

假设我们想循环一个待办列表，只显示未完成的任务：

```html
<!-- ❌ 错误：v-if先执行，无法访问todo变量 -->
<li v-for="todo in todos" v-if="!todo.done" :key="todo.id">
    {{ todo.text }}
</li>
```

运行后会报错：`Property 'todo' was accessed during render but is not defined on instance.`  
原因很简单：`v-if`执行时，`v-for`还未开始循环，`todo`变量根本不存在。

### 最佳实践：先过滤，再循环

既然不能直接在同一个元素上混用，我们需要**分离“过滤逻辑”和“循环逻辑”**——用**计算属性（computed）**先过滤数据，再用`v-for`
循环过滤后的结果。

#### 示例：过滤未完成的待办任务

```html

<template>
    <div class="todo-list">
        <h2>未完成的任务</h2>
        <ul>
            <!-- 循环过滤后的计算属性 -->
            <li v-for="todo in incompleteTodos" :key="todo.id">
                {{ todo.text }}
            </li>
        </ul>
    </div>
</template>

<script setup>
    import {ref, computed} from 'vue'

    // 原始待办数据（响应式）
    const todos = ref([
        {id: 1, text: '学习Vue3优先级规则', done: false},
        {id: 2, text: '写一篇技术博客', done: true},
        {id: 3, text: '锻炼30分钟', done: false}
    ])

    // 计算属性：过滤未完成的任务（自动缓存）
    const incompleteTodos = computed(() => {
        return todos.value.filter(todo => !todo.done)
    })
</script>
```

#### 为什么用计算属性？

1. **解决优先级冲突**：计算属性先处理数据过滤，`v-for`直接循环过滤后的结果，无需再用`v-if`。
2. **性能优化**：计算属性会缓存结果，只有当依赖的`todos`变化时才会重新计算（避免重复过滤）。
3. **逻辑清晰**：过滤逻辑封装在计算属性中，模板更简洁，维护成本更低。

### 流程梳理：从数据到渲染的完整链路

为了更直观理解，我们用**流程图**展示“过滤→循环→渲染”的过程：

```mermaid
graph TD
    A[原始数据：todos数组] --> B[计算属性：incompleteTodos = todos.value.filter（todo => !todo.done）]
    B --> C[v-for循环incompleteTodos]
    C --> D[渲染未完成的任务]
```

### 扩展场景：多条件过滤与分组

如果需要根据多个条件过滤（比如“未完成且优先级高”），或分组显示（比如“已完成”和“未完成”分开），计算属性依然是最佳选择。

#### 示例：分组显示待办任务

```html

<template>
    <div class="todo-group">
        <h3>未完成任务（{{ incompleteTodos.length }}）</h3>
        <ul class="incomplete">
            <li v-for="todo in incompleteTodos" :key="todo.id">{{ todo.text }}</li>
        </ul>

        <h3>已完成任务（{{ completedTodos.length }}）</h3>
        <ul class="completed">
            <li v-for="todo in completedTodos" :key="todo.id">{{ todo.text }}</li>
        </ul>
    </div>
</template>

<script setup>
    import {ref, computed} from 'vue'

    const todos = ref([/* 同前 */])

    // 未完成任务（过滤条件：done === false）
    const incompleteTodos = computed(() => todos.value.filter(t => !t.done))
    // 已完成任务（过滤条件：done === true）
    const completedTodos = computed(() => todos.value.filter(t => t.done))
</script>

<style scoped>
    .incomplete {
        color: #ff4444;
    }

    .completed {
        color: #00cc00;
        text-decoration: line-through;
    }
</style>
```

### 课后Quiz：巩固你的理解

**问题**：在Vue3中，需要循环“商品列表”并只显示“库存>0”的商品时，为什么推荐用计算属性而非直接混用`v-if`和`v-for`？请写出实现代码。

**答案解析**：

1. **原因**：
    - 优先级冲突：`v-if`先执行会导致无法访问`v-for`的循环变量（如`product`）；
    - 性能浪费：直接混用会循环所有商品再隐藏无库存的，而计算属性先过滤，减少渲染量；
    - 可维护性：过滤逻辑封装在计算属性中，模板更简洁。

2. **实现代码**：

```html

<template>
    <div class="product-list">
        <h2>有库存的商品</h2>
        <ul>
            <li v-for="p in availableProducts" :key="p.id">
                {{ p.name }} - 库存：{{ p.stock }}
            </li>
        </ul>
    </div>
</template>

<script setup>
    import {ref, computed} from 'vue'

    const products = ref([
        {id: 1, name: 'Vue3实战', stock: 5},
        {id: 2, name: 'React入门', stock: 0},
        {id: 3, name: 'TS手册', stock: 3}
    ])

    // 计算属性：过滤库存>0的商品
    const availableProducts = computed(() => {
        return products.value.filter(p => p.stock > 0)
    })
</script>
```

### 常见报错及解决

#### 1. 错误：`Property 'item' was accessed during render but is not defined`

- **原因**：同一个元素上混用`v-if`和`v-for`，`v-if`先执行导致无法访问`v-for`的循环变量。
- **解决**：用计算属性过滤数据，或把`v-for`放到父级`<template>`标签上：
  ```html
  <template v-for="todo in todos" :key="todo.id">
    <li v-if="!todo.done">{{ todo.text }}</li>
  </template>
  ```

#### 2. 错误：`Missing required prop: "key"`

- **原因**：`v-for`未指定唯一`key`属性，Vue无法跟踪元素身份。
- **解决**：给`v-for`的元素添加`:key`，值为唯一标识（如`id`）：
  ```html
  <li v-for="todo in incompleteTodos" :key="todo.id">{{ todo.text }}</li>
  ```

#### 3. 错误：`Computed property 'filteredList' has no setter`

- **原因**：试图修改计算属性的值，但计算属性默认只读（只有`getter`）。
- **解决**：如需修改，定义`setter`（或直接修改原始数据）：
  ```js
  const filteredList = computed({
    get() { return list.value.filter(i => i.active) },
    set(newValue) { list.value = newValue.map(i => ({ ...i, active: true })) }
  })
  ```
