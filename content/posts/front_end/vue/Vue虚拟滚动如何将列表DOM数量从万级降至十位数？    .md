### 大规模列表渲染的性能痛点

当我们用`v-for`
渲染10000条以上数据时，页面会生成大量DOM节点——每个节点都要占用内存，浏览器的渲染、重排、重绘成本会指数级上升。常见症状包括：页面初始化慢（白屏时间长）、滚动卡顿（掉帧）、交互响应延迟（点击按钮没反应）。这些问题的核心是
**DOM数量过多**，需要针对性优化。

### 技巧一：虚拟滚动——只渲染“看得见”的内容

虚拟滚动（Virtual Scrolling）是解决大规模列表性能问题的核心方案。它的本质是**仅渲染当前可视区域内的列表项**
，滚动时动态替换内容，同时用占位元素保持滚动条的正常长度。

#### 虚拟滚动的原理

实现虚拟滚动需要三个关键计算：

1. **可视区域高度**：容器的可见高度（比如`500px`）；
2. **单条项高度**：每个列表项的固定高度（比如`50px`）；
3. **滚动位置**：滚动条的偏移量（比如`scrollTop = 200px`）。

通过这三个值，可以算出：

- 可视区域能容纳的项数：`500 / 50 = 10`条；
- 起始项索引：`Math.floor(200 / 50) = 4`（从第5条开始）；
- 结束项索引：`4 + 10 = 14`（到第15条结束）。

最终，只渲染这10条项，其余项用占位元素填充，从而将DOM数量从10000降到10。

#### 实战：用`vue-virtual-scroller`实现虚拟滚动

Vue生态中最成熟的虚拟滚动库是`vue-virtual-scroller`（支持Vue 3），我们用它快速实现。

**步骤1：安装依赖**

```bash
npm install vue-virtual-scroller@next  # @next 对应Vue 3版本（当前最新v2.0.0-beta.7）
```

**步骤2：编写组件**

```vue

<template>
  <div class="virtual-list">
    <!-- RecycleScroller 是核心组件 -->
    <RecycleScroller
        class="scroller"
        :items="largeList"       <!-- 全部列表数据（10000条） -->
    :item-size="50"          <!-- 每条项的固定高度 -->
    key-field="id"           <!-- 项的唯一标识（必填） -->
    v-slot="{ item }"        <!-- 插槽传递当前渲染的项 -->
    >
    <div class="list-item">
      {{ item.id }} - {{ item.content }}
    </div>
    </RecycleScroller>
  </div>
</template>

<script setup>
  import {ref} from 'vue';
  import {RecycleScroller} from 'vue-virtual-scroller';  // 引入组件
  import 'vue-virtual-scroller/dist/vue-virtual-scroller.css';  // 引入样式

  // 生成10000条测试数据
  const largeList = ref(Array.from({length: 10000}, (_, i) => ({
    id: i,
    content: `Item ${i}`
  })));
</script>

<style scoped>
  .virtual-list {
    width: 300px;
    height: 500px; /* 容器固定高度，开启滚动 */
    border: 1px solid #eee;
  }

  .scroller {
    height: 100%; /* 让 RecycleScroller 填满容器 */
  }

  .list-item {
    height: 50px; /* 必须和 item-size 一致 */
    line-height: 50px;
    padding: 0 16px;
    border-bottom: 1px solid #f0f0f0;
  }
</style>
```

### 技巧二：`v-memo`——缓存模板片段，避免重复渲染

`v-memo`是Vue 3新增的**模板缓存指令**，用于复用未变化的模板片段。当依赖的响应式数据不变时，直接跳过渲染，从而减少性能开销。

#### 基本用法

`v-memo`接受一个**依赖数组**，格式如下：

```vue

<div v-memo="[dep1, dep2]">
  {{ dep1 }} - {{ dep2 }}
</div>
```  

- 当依赖数组中的任意值变化时，重新渲染；
- 否则，复用缓存的内容。

#### 结合`v-for`优化列表项

`v-memo`最常用的场景是和`v-for`结合，缓存每个列表项：

```vue

<template>
  <div class="product-list">
    <div
        v-for="product in products"
        :key="product.id"
    <!-- 依赖数组：仅product.id或product.name变化时重新渲染 -->
    v-memo="[product.id, product.name]"
    class="product-item"
    >
    {{ product.id }} - {{ product.name }}
  </div>
  </div>
</template>

<script setup>
  import {ref} from 'vue';

  const products = ref(Array.from({length: 1000}, (_, i) => ({
    id: i,
    name: `Product ${i}`,
    // 其他不影响渲染的字段（比如库存）
    stock: 100
  })));

  // 模拟更新一个商品名称
  setTimeout(() => {
    products.value[500].name = 'Updated Product';
  }, 2000);
</script>
```

### 课后Quiz：巩固你的理解

#### 问题1：虚拟滚动为什么能提升性能？

**答案**：虚拟滚动只渲染可视区域的项，将DOM数量从“万级”降到“十级”，大幅减少浏览器的渲染和内存开销。

#### 问题2：`v-memo`的依赖数组为空（`v-memo="[]"`）时，会发生什么？

**答案**：模板片段会被**永久缓存**，除非组件本身重新渲染（比如父组件传递的props变化）。因为依赖数组为空意味着没有需要监听的变化，所以不会触发更新。

### 常见报错及解决方案

#### 1. 虚拟滚动时滚动条跳动

**错误表现**：滚动时列表项突然“跳位”，或滚动条长度异常。  
**原因**：`item-size`设置的高度与列表项实际高度不一致（比如`item-size=50`，但实际项高是`60px`）。  
**解决**：

- 确保`item-size`与项的实际高度完全一致；
- 如果项高度不固定，开启`dynamic-item-size`属性：
  ```vue
  <RecycleScroller :dynamic-item-size="true" />
  ```  

#### 2. `v-memo`后数据更新但界面不变化

**错误表现**：修改了`product.name`，但界面没更新。  
**原因**：`v-memo`的依赖数组没有包含`product.name`，导致Vue认为不需要更新。  
**解决**：将所有影响渲染的字段加入依赖数组：

```vue
<!-- 错误：依赖数组缺少product.name -->
v-memo="[product.id]"
<!-- 正确：包含所有影响渲染的字段 -->
v-memo="[product.id, product.name]"
```  

#### 3. 安装`vue-virtual-scroller`提示“模块未找到”

**错误表现**：`Cannot find module 'vue-virtual-scroller'`  
**原因**：未安装依赖或安装失败。  
**解决**：

- 重新安装：`npm install vue-virtual-scroller@next`；
- 检查`package.json`的`dependencies`中是否有`vue-virtual-scroller`，没有则手动添加后重新安装。
