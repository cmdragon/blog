## 一、Props 基础入门：从声明到传递

### 1.1 Props 声明的三种方式

Props 是 Vue 组件间通信的核心方式之一，组件必须显式声明接收的 Props，Vue 才能正确识别外部传入的属性。以下是三种常用的声明方式：

#### 数组式声明（简单场景）

适合不需要类型校验的基础场景：

```vue

<script setup>
  // 声明接收 foo 属性
  const props = defineProps(['foo'])
  console.log(props.foo) // 访问传入的 foo 值
</script>
```

#### 对象式声明（带类型校验）

支持类型校验、默认值等高级配置，是生产环境的推荐方式：

```vue

<script setup>
  defineProps({
    title: String, // 字符串类型
    likes: {
      type: Number,
      default: 0, // 默认值
      required: true // 必填项
    },
    isPublished: Boolean
  })
</script>
```

#### TypeScript 类型注解（类型安全）

使用 TypeScript 时可以直接通过类型声明 Props，兼顾类型安全和简洁性：

```vue

<script setup lang="ts">
  defineProps<{
    title?: string // 可选字符串
    likes: number // 必填数字
    tags?: string[] // 可选字符串数组
  }>()
</script>
```

### 1.2 响应式 Props 解构的正确姿势

Vue 3.5+ 对 Props 解构做了优化，自动保留响应式，但仍有一些注意事项：

#### 基础解构（Vue 3.5+）

```vue

<script setup>
  // 解构后自动保留响应式
  const {title, likes} = defineProps < {title: string; likes: number} > ()

  // Vue 编译器会自动转换为 props.title，保持响应式
  watchEffect(() => {
    console.log(`文章标题：${title}，点赞数：${likes}`)
  })
</script>
```

#### 传递解构后的 Props 到函数

直接传递解构值会丢失响应式，需要用 getter 包裹：

```vue

<script setup>
  const {likes} = defineProps < {likes: number} > ()

  // ❌ 错误：传递的是当前值，不是响应式源
  // watch(likes, (newVal) => console.log(newVal))

  // ✅ 正确：用 getter 包裹保留响应式
  watch(() => likes, (newVal) => console.log(`点赞数更新为：${newVal}`))
</script>
```

### 1.3 Props 传递的细节规范

#### 命名规范：camelCase vs kebab-case

- 组件内部声明用 camelCase（符合 JavaScript 习惯）
- 父组件传递用 kebab-case（符合 HTML 习惯）

```vue
<!-- 子组件声明 -->
<script setup>
  defineProps({greetingMessage: String})
</script>

<!-- 父组件传递 -->
<ChildComponent greeting-message="Hello Vue 3!"/>
```

#### 不同类型值的传递方式

```vue
<!-- 静态字符串 -->
<BlogPost title="Vue 3 入门指南"/>

<!-- 动态绑定变量 -->
<BlogPost :title="post.title" :likes="post.likes"/>

<!-- 布尔类型：省略值表示 true -->
<BlogPost is-published/>
<BlogPost :is-published="false"/> <!-- 显式传递 false -->

<!-- 数组和对象：必须用 v-bind -->
<BlogPost :tags="['Vue', 'JavaScript']"/>
<BlogPost :author="{ name: '张三', age: 28 }"/>

<!-- 批量传递对象属性 -->
<BlogPost v-bind="post"/> <!-- 等价于 :id="post.id" :title="post.title" -->
```

## 二、Props 核心原理与最佳实践

### 2.1 单向数据流：为什么不能在子组件修改 Props？

Vue 中 Props 是单向绑定的：父组件更新会自动同步到子组件，但子组件不能直接修改 Props。这样设计的目的是：

1. 避免组件间数据流向混乱，提升可维护性
2. 防止子组件意外修改父组件状态，导致调试困难

#### 两种常见场景的正确处理方式

**场景1：Props 作为初始值，子组件需要独立修改**
将 Props 赋值给本地响应式变量：

```vue

<script setup>
  const props = defineProps({initialCount: {type: Number, default: 0}})
  // 本地变量，与 Props 解绑
  const count = ref(props.initialCount)
</script>
```

**场景2：Props 需要转换后使用**
使用 computed 派生响应式数据：

```vue

<script setup>
  const props = defineProps({size: String})
  // 自动同步 Props 变化
  const normalizedSize = computed(() => props.size.trim().toLowerCase())
</script>
```

### 2.2 复杂类型 Props 的注意事项

当传递对象或数组时，子组件虽然不能修改 Props 引用，但可以修改其内部属性：

```vue

<script setup>
  const props = defineProps({user: Object})

  // ⚠️ 不推荐：直接修改对象内部属性，父组件无法感知
  props.user.name = '李四'

  // ✅ 推荐：通过事件通知父组件修改
  const emit = defineEmits(['updateUser'])
  const updateUserName = () => {
    emit('updateUser', {...props.user, name: '李四'})
  }
</script>
```

### 2.3 Prop 验证：提前拦截错误

Prop 验证可以在开发阶段提前发现数据类型错误，提升组件健壮性：

```vue

<script setup>
  defineProps({
    // 基础类型校验
    id: Number,
    // 多类型支持
    status: [String, Number],
    // 必填项 + 自定义验证
    level: {
      type: String,
      required: true,
      validator: (value) => {
        return ['primary', 'success', 'warning', 'danger'].includes(value)
      }
    },
    // 对象默认值（工厂函数）
    config: {
      type: Object,
      default: () => ({theme: 'light', size: 'medium'})
    }
  })
</script>
```

## 三、Props 性能优化技巧

### 3.1 避免传递不必要的 Props

只传递子组件真正需要的属性，减少不必要的重渲染：

```vue
<!-- ❌ 不推荐：传递整个对象 -->
<UserCard :user="user"/>

<!-- ✅ 推荐：只传递需要的属性 -->
<UserCard :name="user.name" :avatar="user.avatar" :level="user.level"/>
```

### 3.2 使用 `memo` 缓存组件

当子组件接收复杂类型 Props 时，使用 `memo` 包裹组件，只有当 Props 真正变化时才重渲染：

```vue

<script setup>
  import {memo} from 'vue'

  const UserCard = memo((props) => {
    return <div>{props.name}</div>
  })
</script>
```

### 3.3 对大体积 Props 使用 `shallowRef`

如果传递的对象不需要深度响应式，可以用 `shallowRef` 包装，提升性能：

```vue

<script setup>
  import {shallowRef} from 'vue'

  // 大数据对象，不需要深度响应
  const bigData = shallowRef({ /* 大量数据 */})
</script>

<BigDataComponent :data="bigData"/>
```

## 四、课后 Quiz：巩固你的 Props 知识

### 问题1：以下代码有什么问题？如何修复？

```vue

<script setup>
  const {count} = defineProps({count: Number})
  // 尝试监听解构后的 count
  watch(count, (newVal) => console.log(newVal))
</script>
```

**答案解析**：
问题：解构后的 `count` 是普通值，不是响应式数据源，`watch` 无法监听变化。
修复方式：用 getter 包裹：

```vue
watch(() => count, (newVal) => console.log(newVal))
```

### 问题2：子组件需要修改父组件传递的对象 Props，正确的做法是什么？

**答案解析**：
子组件不应该直接修改对象内部属性，应该通过事件通知父组件修改：

```vue
<!-- 子组件 -->
<script setup>
  const props = defineProps({user: Object})
  const emit = defineEmits(['updateUser'])

  const updateUser = () => {
    emit('updateUser', {...props.user, age: props.user.age + 1})
  }
</script>

<!-- 父组件 -->
<ChildComponent :user="user" @update-user="user = $event"/>
```

## 五、常见报错与解决方案

### 报错1：`Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders.`

**原因**：子组件直接修改了 Props 值，违反单向数据流原则。
**解决方案**：

- 如果是需要独立修改的值，将 Props 赋值给本地响应式变量
- 如果是需要转换的值，使用 computed 派生数据
- 如果必须修改父组件状态，通过 emit 事件通知父组件

### 报错2：`Invalid prop: type check failed for prop "xxx". Expected String, got Number.`

**原因**：传递的 Props 类型与组件声明的类型不匹配。
**解决方案**：

- 检查父组件传递的值类型是否正确
- 如果需要支持多类型，在组件声明时使用数组形式：`type: [String, Number]`

### 报错3：`Missing required prop: "xxx"`

**原因**：组件声明了必填 Props，但父组件没有传递。
**解决方案**：

- 父组件补充传递该 Props
- 如果不是必须的，将组件中的 `required: true` 移除，或设置默认值
