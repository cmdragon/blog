## 一、v-drag：拖拽指令，让元素随心移动

拖拽这个功能，说起来简单——鼠标按下、移动、松开，三步走。但真要写好，还是有不少细节要处理的。

### 1.1 拖拽的核心思路

想象一下你在桌面上拖一个便签纸：先用手指按住它，然后移动手指，便签就跟着走，松手就停。DOM元素的拖拽也是这个道理，只不过"手指"变成了鼠标事件：

- `mousedown`：手指按下去
- `mousemove`：手指移动中
- `mouseup`：手指松开了

整个流程可以用下面这个图来理解：

```
用户按下鼠标(mousedown)
    │
    ├─ 记录起始位置(startX, startY)
    ├─ 记录元素当前位置(initialLeft, initialTop)
    │
    ▼
用户移动鼠标(mousemove)
    │
    ├─ 计算偏移量 dx = clientX - startX
    ├─ 计算偏移量 dy = clientY - startY
    ├─ 设置新位置 left = initialLeft + dx
    ├─ 设置新位置 top = initialTop + dy
    │
    ▼
用户松开鼠标(mouseup)
    │
    └─ 停止拖拽(isDragging = false)
```

### 1.2 完整代码实现

```javascript
const vDrag = {
  mounted(el, binding) {
    // 设置元素为可定位、可移动的样式
    el.style.cursor = "move";
    el.style.position = "absolute";

    let isDragging = false;
    let startX, startY, initialLeft, initialTop;

    const onMouseDown = (e) => {
      isDragging = true;
      // 记录鼠标按下时的坐标
      startX = e.clientX;
      startY = e.clientY;
      // 记录元素当前的偏移位置
      initialLeft = el.offsetLeft;
      initialTop = el.offsetTop;
      // 阻止默认行为，防止拖拽时选中文字
      e.preventDefault();
    };

    const onMouseMove = (e) => {
      if (!isDragging) return;
      // 计算鼠标移动的距离
      const dx = e.clientX - startX;
      const dy = e.clientY - startY;
      // 更新元素位置
      el.style.left = initialLeft + dx + "px";
      el.style.top = initialTop + dy + "px";
    };

    const onMouseUp = () => {
      isDragging = false;
    };

    // mousedown绑定在元素自身
    el.addEventListener("mousedown", onMouseDown);
    // mousemove和mouseup绑定在document上（后面Quiz会解释为啥）
    document.addEventListener("mousemove", onMouseMove);
    document.addEventListener("mouseup", onMouseUp);

    // 把清理函数挂到元素上，unmounted时调用
    el._dragCleanup = () => {
      el.removeEventListener("mousedown", onMouseDown);
      document.removeEventListener("mousemove", onMouseMove);
      document.removeEventListener("mouseup", onMouseUp);
    };
  },
  unmounted(el) {
    // 组件卸载时清理事件监听，防止内存泄漏
    el._dragCleanup?.();
  },
};
```

### 1.3 限制拖拽范围

有时候你不希望元素被拖出某个容器，比如一个弹窗只能在页面内移动。这时候可以通过`binding.value`传入限制参数：

```javascript
const vDrag = {
  mounted(el, binding) {
    el.style.cursor = "move";
    el.style.position = "absolute";

    let isDragging = false;
    let startX, startY, initialLeft, initialTop;

    const onMouseDown = (e) => {
      isDragging = true;
      startX = e.clientX;
      startY = e.clientY;
      initialLeft = el.offsetLeft;
      initialTop = el.offsetTop;
      e.preventDefault();
    };

    const onMouseMove = (e) => {
      if (!isDragging) return;
      let dx = e.clientX - startX;
      let dy = e.clientY - startY;
      let newLeft = initialLeft + dx;
      let newTop = initialTop + dy;

      // 如果传了边界限制，就做边界判断
      if (binding.value) {
        const { minX, maxX, minY, maxY } = binding.value;
        if (minX !== undefined) newLeft = Math.max(minX, newLeft);
        if (maxX !== undefined) newLeft = Math.min(maxX, newLeft);
        if (minY !== undefined) newTop = Math.max(minY, newTop);
        if (maxY !== undefined) newTop = Math.min(maxY, newTop);
      }

      el.style.left = newLeft + "px";
      el.style.top = newTop + "px";
    };

    const onMouseUp = () => {
      isDragging = false;
    };

    el.addEventListener("mousedown", onMouseDown);
    document.addEventListener("mousemove", onMouseMove);
    document.addEventListener("mouseup", onMouseUp);

    el._dragCleanup = () => {
      el.removeEventListener("mousedown", onMouseDown);
      document.removeEventListener("mousemove", onMouseMove);
      document.removeEventListener("mouseup", onMouseUp);
    };
  },
  unmounted(el) {
    el._dragCleanup?.();
  },
};
```

使用的时候这样传参：

```html
<!-- 不限制范围 -->
<div v-drag>随便拖</div>

<!-- 限制在0~500的范围内 -->
<div v-drag="{ minX: 0, maxX: 500, minY: 0, maxY: 400 }">只能在这个框里拖</div>
```

### 1.4 使用效果

在模板里给元素加上`v-drag`，这个元素就能被鼠标拖来拖去了。是不是贼简单？以前写拖拽还得搞一堆状态变量，现在一个指令搞定，代码清爽多了。

## 二、v-copy：一键复制指令，点击就复制文本

做后台管理系统的时候，"复制"这个操作出现的频率贼高——复制订单号、复制邀请链接、复制分享码……每次都要写`navigator.clipboard.writeText`，写多了真的烦。干脆封装成指令，一劳永逸。

### 2.1 复制的核心思路

浏览器的`navigator.clipboard.writeText()`API就是专门干这事儿的，调用它就能把文本写进剪贴板。但有几个注意点：

- 这个API需要HTTPS环境，HTTP下会报错
- 调用后返回Promise，需要处理成功和失败
- 要给用户一个反馈，不然用户不知道到底复制没复制

### 2.2 完整代码实现

```javascript
const vCopy = {
  mounted(el, binding) {
    el.style.cursor = "pointer";
    // 把要复制的文本存起来
    el._copyText = binding.value;

    const copy = async () => {
      const text = el._copyText;
      if (!text) {
        console.warn("v-copy: 没有提供要复制的文本");
        return;
      }

      try {
        await navigator.clipboard.writeText(text);
        // 复制成功，给个视觉反馈
        showCopyFeedback(el, "复制成功！");
      } catch (err) {
        // clipboard API失败，降级用老方法
        try {
          const textarea = document.createElement("textarea");
          textarea.value = text;
          textarea.style.position = "fixed";
          textarea.style.opacity = "0";
          document.body.appendChild(textarea);
          textarea.select();
          document.execCommand("copy");
          document.body.removeChild(textarea);
          showCopyFeedback(el, "复制成功！");
        } catch (fallbackErr) {
          showCopyFeedback(el, "复制失败", true);
          console.error("v-copy: 复制失败", fallbackErr);
        }
      }
    };

    el.addEventListener("click", copy);
    el._copyCleanup = () => {
      el.removeEventListener("click", copy);
    };
  },
  updated(el, binding) {
    // 指令绑定的值变了，更新要复制的文本
    el._copyText = binding.value;
  },
  unmounted(el) {
    el._copyCleanup?.();
  },
};

// 显示复制反馈的小工具函数
function showCopyFeedback(el, message, isError = false) {
  const tip = document.createElement("span");
  tip.textContent = message;
  tip.style.cssText = `
    position: absolute;
    top: -30px;
    left: 50%;
    transform: translateX(-50%);
    padding: 4px 12px;
    border-radius: 4px;
    font-size: 12px;
    color: #fff;
    background: ${isError ? "#f56c6c" : "#67c23a"};
    white-space: nowrap;
    pointer-events: none;
    transition: opacity 0.3s;
  `;
  el.style.position = el.style.position || "relative";
  el.appendChild(tip);

  // 1.5秒后自动消失
  setTimeout(() => {
    tip.style.opacity = "0";
    setTimeout(() => tip.remove(), 300);
  }, 1500);
}
```

### 2.3 使用示例

```html
<template>
  <div>
    <!-- 复制固定文本 -->
    <span v-copy="'这是要复制的内容'">点击复制文本</span>

    <!-- 复制动态数据 -->
    <span v-copy="orderNo">订单号：{{ orderNo }}</span>

    <!-- 复制链接 -->
    <button v-copy="shareLink">复制分享链接</button>
  </div>
</template>

<script setup>

  const shareLink = ref("https://example.com/invite/abc123");
</script>
```

点击元素后，文本就悄悄进剪贴板了，还会弹出一个小提示告诉用户"复制成功"。这个反馈很关键，不然用户点了也不知道到底复制没复制，那可不就白忙活了嘛。

### 2.4 降级方案说明

代码里有个`document.execCommand('copy')`的降级方案，这是给那些不支持`navigator.clipboard`的老浏览器或者HTTP环境准备的。虽然`execCommand`已经被标记为废弃了，但作为兜底方案还是挺好使的，总比直接报错强。

## 三、v-longpress：长按指令，按住不放才触发

长按这个交互在移动端特别常见——长按删除消息、长按保存图片、长按弹出操作菜单。但浏览器原生并没有"长按"事件，需要咱们自己用`setTimeout`来模拟。

### 3.1 长按的核心思路

区分"点击"和"长按"的关键就是时间——按下后等一段时间（比如500ms），如果还没松手，就算长按；如果中途松手了，那就是普通点击。

```
手指/鼠标按下
    │
    ├─ 启动定时器（默认500ms）
    │       │
    │       ├─ 时间到 → 触发长按回调 ✓
    │       │
    │       └─ 中途松手 → 清除定时器，不触发 ✗
    │
    ▼
手指/鼠标松开
    │
    └─ 清除定时器
```

### 3.2 完整代码实现

```javascript
const vLongpress = {
  mounted(el, binding) {
    // binding.value是长按触发的回调函数
    if (typeof binding.value !== "function") {
      console.warn("v-longpress: 绑定的值必须是一个函数");
      return;
    }

    // 支持自定义时长，通过binding.arg或者modifiers来传
    // 用法：v-longpress:800="handleLongpress" 表示800ms
    const duration = binding.arg ? parseInt(binding.arg) : 500;

    let timer = null;

    const start = (e) => {
      // 防止重复触发
      if (timer) clearTimeout(timer);
      timer = setTimeout(() => {
        binding.value(e);
      }, duration);
    };

    const cancel = () => {
      if (timer) {
        clearTimeout(timer);
        timer = null;
      }
    };

    // PC端鼠标事件
    el.addEventListener("mousedown", start);
    el.addEventListener("mouseup", cancel);
    el.addEventListener("mouseleave", cancel);

    // 移动端触摸事件
    el.addEventListener("touchstart", start, { passive: true });
    el.addEventListener("touchend", cancel);
    el.addEventListener("touchcancel", cancel);

    // 阻止长按时的右键菜单（移动端）
    el.addEventListener("contextmenu", (e) => {
      e.preventDefault();
    });

    el._longpressCleanup = () => {
      el.removeEventListener("mousedown", start);
      el.removeEventListener("mouseup", cancel);
      el.removeEventListener("mouseleave", cancel);
      el.removeEventListener("touchstart", start);
      el.removeEventListener("touchend", cancel);
      el.removeEventListener("touchcancel", cancel);
      if (timer) clearTimeout(timer);
    };
  },
  unmounted(el) {
    el._longpressCleanup?.();
  },
};
```

### 3.3 使用示例

```html
<template>
  <div>
    <!-- 默认500ms长按 -->
    <button v-longpress="handleLongpress">长按我试试</button>

    <!-- 自定义800ms长按 -->
    <button v-longpress:800="handleDelete">长按删除</button>

    <!-- 移动端长按保存图片 -->
    <img v-longpress:1000="saveImage" src="/photo.jpg" alt="长按保存" />
  </div>
</template>

<script setup>
  const handleLongpress = () => {
    alert("你长按了！");
  };

  const handleDelete = () => {
    console.log("执行删除操作...");
  };

  const saveImage = () => {
    console.log("保存图片到相册...");
  };
</script>
```

这里有个细节——移动端同时绑定了`touch`事件和`contextmenu`的阻止。为啥呢？因为手机上长按会弹出系统菜单（复制/粘贴那玩意儿），不阻止的话你的长按逻辑还没执行，系统菜单先蹦出来了，那体验可太差了。

## 四、v-permission：权限控制指令，没有权限直接隐藏

做后台系统的时候，权限控制是绕不开的话题。有些按钮只有管理员能看，有些菜单只有特定角色才能操作。用`v-if`当然也能做，但如果权限判断逻辑散落在各个组件里，维护起来那叫一个痛苦。封装成指令，一行代码搞定权限控制，岂不美哉？

### 4.1 权限控制的核心思路

思路很简单：拿到当前用户的权限列表，跟指令传入的权限做比对，匹配得上就保留元素，匹配不上就直接把元素从DOM里移除。注意，是移除，不是`display: none`隐藏——没有权限的元素压根就不应该出现在页面上，这样更安全。

```
指令绑定值(需要的权限)
    │
    ▼
获取当前用户权限列表
    │
    ▼
判断用户是否拥有所需权限
    │
    ├─ 有权限 → 保留元素 ✓
    │
    └─ 无权限 → 移除元素 ✗
```

### 4.2 完整代码实现

```javascript
// 权限列表的获取方式，根据你的项目来调整
// 这里用一个简单的全局变量模拟，实际项目中一般从store或token里取
let currentUserPermissions = [];

// 提供一个设置权限的方法，在登录成功后调用
export function setPermissions(permissions) {
  currentUserPermissions = permissions;
}

// 提供一个获取权限的方法
export function getPermissions() {
  return currentUserPermissions;
}

const vPermission = {
  mounted(el, binding) {
    const requiredPermission = binding.value;
    const userPermissions = currentUserPermissions;

    if (!requiredPermission) return;

    let hasPermission = false;

    if (Array.isArray(requiredPermission)) {
      // 权限数组：用户拥有其中任意一个即可（OR逻辑）
      hasPermission = requiredPermission.some((perm) =>
        userPermissions.includes(perm),
      );
    } else if (typeof requiredPermission === "string") {
      // 单个权限：直接判断
      hasPermission = userPermissions.includes(requiredPermission);
    }

    if (!hasPermission) {
      // 没有权限，直接从DOM中移除元素
      el.parentNode?.removeChild(el);
    }
  },
};
```

### 4.3 使用示例

```html
<template>
  <div>
    <!-- 单个权限控制：只有admin角色能看到 -->
    <button v-permission="'admin'">删除用户</button>

    <!-- 权限数组：admin或editor都能看到 -->
    <button v-permission="['admin', 'editor']">编辑文章</button>

    <!-- 结合动态权限 -->
    <div v-permission="userRole">管理员专属区域</div>
  </div>
</template>

<script setup>
  import { ref, onMounted } from "vue";
  import { setPermissions } from "./directives/permission";

  const userRole = ref("editor");

  onMounted(() => {
    // 模拟登录后设置权限
    // 实际项目中从接口获取
    setPermissions(["editor", "viewer"]);
  });
</script>
```

### 4.4 和v-if的对比

有人可能会问：这玩意儿用`v-if`不也能实现吗？确实能，但自定义指令有几个优势：

- **逻辑集中**：权限判断逻辑写在指令里，不用每个组件都写一遍
- **使用简洁**：模板里一行`v-permission="'admin'"`就搞定，不用引入computed或者store
- **更安全**：元素直接从DOM移除，比`v-if`的注释占位更彻底，审查元素也看不到
- **可维护**：权限逻辑改了只改指令一处，所有用到的地方自动生效

当然，如果你的项目权限逻辑特别复杂（比如需要动态更新、需要缓存等），用`v-if`配合组合式函数可能更灵活。选哪个看你的具体场景，别死磕。

## 五、v-lazyload：图片懒加载指令，滚动到可视区域才加载

一个页面放了几十张图片，如果一打开就全部加载，那流量和性能都是灾难。懒加载的思路就是——图片先不加载，等用户滚动到它附近了再加载。这玩意儿用`IntersectionObserver`来实现简直不要太方便。

### 5.1 懒加载的核心思路

`IntersectionObserver`是浏览器提供的一个API，它可以帮你"观察"一个元素是否进入了视口。你只需要告诉它"帮我盯着这个元素"，等元素进入视口了它就会通知你。

```
页面加载
    │
    ├─ 所有图片先用占位符(data-src存真实地址)
    │
    ▼
IntersectionObserver开始监听
    │
    ▼
用户滚动页面
    │
    ├─ 图片进入视口？
    │     │
    │     ├─ 是 → 把data-src赋值给src，加载真实图片
    │     │       加载完成后添加过渡动画
    │     │       停止观察该图片
    │     │
    │     └─ 否 → 继续等待
    │
    ▼
所有图片加载完毕 → 断开Observer
```

### 5.2 完整代码实现

```javascript
// 默认占位图（1x1透明gif）
const DEFAULT_PLACEHOLDER =
  "data:image/gif;base64,R0lGODlhAQABAIAAAAAAAP///yH5BAEAAAAALAAAAAABAAEAAAIBRAA7";

const vLazyload = {
  mounted(el, binding) {
    // 获取占位图，没传就用默认的
    const placeholder = binding.arg || DEFAULT_PLACEHOLDER;
    // 真实图片地址从binding.value取
    const realSrc = binding.value;

    if (!realSrc) return;

    // 先设置占位图
    el.src = placeholder;
    // 初始透明，加载完成后做渐显动画
    el.style.opacity = "0";
    el.style.transition = "opacity 0.3s ease";

    // 创建IntersectionObserver
    const observer = new IntersectionObserver(
      (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            // 元素进入视口，加载真实图片
            const img = entry.target;
            img.src = realSrc;

            img.onload = () => {
              // 加载完成，渐显
              img.style.opacity = "1";
            };

            img.onerror = () => {
              // 加载失败也显示出来（至少让用户知道图挂了）
              img.style.opacity = "1";
              console.warn("v-lazyload: 图片加载失败", realSrc);
            };

            // 已经加载了，不需要再观察
            observer.unobserve(img);
          }
        });
      },
      {
        // 提前200px开始加载，用户滚动时体验更流畅
        rootMargin: "200px 0px",
      },
    );

    observer.observe(el);
    // 保存observer引用，方便卸载时断开
    el._lazyObserver = observer;
  },
  updated(el, binding) {
    // 如果图片地址变了，重新设置
    if (binding.value !== binding.oldValue) {
      el.src = binding.arg || DEFAULT_PLACEHOLDER;
      el.style.opacity = "0";
      el._lazyObserver?.observe(el);
    }
  },
  unmounted(el) {
    // 组件卸载时断开观察
    el._lazyObserver?.disconnect();
  },
};
```

### 5.3 使用示例

```html
<template>
  <div>
    <!-- 基础用法：直接传图片地址 -->
    <img v-lazyload="imageUrl" alt="懒加载图片" />

    <!-- 自定义占位图 -->
    <img v-lazyload:="/placeholder.png" ="imageUrl" alt="自定义占位" />

    <!-- 列表中的懒加载 -->
    <div v-for="item in imageList" :key="item.id">
      <img v-lazyload="item.url" :alt="item.title" />
    </div>
  </div>
</template>

<script setup>
  import { ref } from "vue";

  const imageUrl = ref("https://example.com/photo.jpg");

  const imageList = ref([
    { id: 1, url: "https://example.com/1.jpg", title: "图片1" },
    { id: 2, url: "https://example.com/2.jpg", title: "图片2" },
    { id: 3, url: "https://example.com/3.jpg", title: "图片3" },

```

这里有个`rootMargin: '200px 0px'`的设置，意思是图片距离视口还有200px的时候就开始加载了，不用等到完全进入视口。这样用户滚动的时候图片已经加载好了，不会出现白屏闪烁的情况，体验丝滑得很。

五个指令都写好了，总不能一个一个`app.directive()`注册吧？那也太丑了。咱们来搞个批量注册，顺便加上TypeScript类型定义，让整个结构更规范。

// directives/types.ts
import type { Directive, DirectiveBinding } from "vue";

// 拖拽范围限制
export interface DragBounds {
  minX?: number;
  maxX?: number;
  minY?: number;
  maxY?: number;
}

// 指令映射类型
export interface DirectiveMap {
  drag: Directive<HTMLElement, DragBounds | undefined>;
  copy: Directive<HTMLElement, string>;

### 6.3 各指令文件

每个指令单独一个文件，导出即可。以`drag.ts`为例：

```typescript
// directives/drag.ts
import type { Directive } from "vue";
import type { DragBounds } from "./types";

export const vDrag: Directive<HTMLElement, DragBounds | undefined> = {
  mounted(el, binding) {
    el.style.cursor = "move";
    el.style.position = "absolute";

    let isDragging = false;
    let startX = 0;
    let startY = 0;
    let initialLeft = 0;
    let initialTop = 0;

    const onMouseDown = (e: MouseEvent) => {
      isDragging = true;
      startX = e.clientX;
      startY = e.clientY;
      initialLeft = el.offsetLeft;
      initialTop = el.offsetTop;
      e.preventDefault();
    };

    const onMouseMove = (e: MouseEvent) => {
      if (!isDragging) return;
      let newLeft = initialLeft + (e.clientX - startX);
      let newTop = initialTop + (e.clientY - startY);

      if (binding.value) {
        const { minX, maxX, minY, maxY } = binding.value;
        if (minX !== undefined) newLeft = Math.max(minX, newLeft);
        if (maxX !== undefined) newLeft = Math.min(maxX, newLeft);
        if (minY !== undefined) newTop = Math.max(minY, newTop);
        if (maxY !== undefined) newTop = Math.min(maxY, newTop);
      }

      el.style.left = newLeft + "px";
      el.style.top = newTop + "px";
    };

    const onMouseUp = () => {
      isDragging = false;
    };

    el.addEventListener("mousedown", onMouseDown);
    document.addEventListener("mousemove", onMouseMove);
    document.addEventListener("mouseup", onMouseUp);

    el._dragCleanup = () => {
      el.removeEventListener("mousedown", onMouseDown);
      document.removeEventListener("mousemove", onMouseMove);
      document.removeEventListener("mouseup", onMouseUp);
    };
  },
  unmounted(el) {
    el._dragCleanup?.();
  },
};
```

其他几个指令文件也类似，各自导出自己的指令对象。

### 6.4 统一注册入口

```typescript
// directives/index.ts
import type { App } from "vue";
import { vDrag } from "./drag";
import { vCopy } from "./copy";
import { vLongpress } from "./longpress";
import { vPermission } from "./permission";
import { vLazyload } from "./lazyload";

// 所有指令集合
const directives = {
  drag: vDrag,
  copy: vCopy,
  longpress: vLongpress,
  permission: vPermission,
  lazyload: vLazyload,
};

// 批量注册函数
export function registerDirectives(app: App) {
  Object.entries(directives).forEach(([name, directive]) => {
    app.directive(name, directive);
  });
}

// 也支持单独导出，方便按需使用
export { vDrag, vCopy, vLongpress, vPermission, vLazyload };
```

### 6.5 在main.ts中使用

```typescript
// main.ts
import { createApp } from "vue";
import App from "./App.vue";
import { registerDirectives } from "./directives";

const app = createApp(App);

// 一行搞定所有指令注册
registerDirectives(app);

app.mount("#app");
```

## 课后Quiz

### 问题1：v-drag指令中，为什么mousemove和mouseup要绑定在document上而不是元素本身？

**答案解析**：

想象一下这个场景：你拖着一个元素快速移动，鼠标可能会"跑"到元素外面去。如果`mousemove`绑定在元素上，一旦鼠标离开了元素范围，就收不到`mousemove`事件了，拖拽就会中断——元素"跟不上"鼠标。

绑定在`document`上就不一样了，不管鼠标跑到哪里，只要还在页面上，`mousemove`都能捕获到，拖拽就不会断。`mouseup`同理，用户可能在元素外面松开鼠标，如果绑定在元素上就收不到松手事件，拖拽状态就永远无法重置了。

这是DOM事件处理中一个很经典的技巧，不光拖拽，任何需要"持续追踪鼠标"的场景都适用。

### 问题2：v-permission指令用v-if也能实现，用自定义指令有什么优势？

**答案解析**：

`v-if`确实能实现类似效果，但自定义指令有几个独到之处：

1. **关注点分离**：权限判断逻辑封装在指令里，组件模板只管声明"这个元素需要什么权限"，不用关心权限怎么判断的。`v-if`的话，每个组件都得写判断逻辑。

2. **复用性**：指令注册一次，全项目通用。`v-if`需要在每个组件里引入权限判断函数或者computed。

3. **更彻底的隐藏**：`v-permission`直接从DOM移除元素，连注释节点都不留。`v-if`虽然也不渲染，但会在DOM里留下注释占位符`<!--v-if-->`。

4. **维护成本低**：权限逻辑改了，只改指令一处。用`v-if`的话，得把所有组件里的判断逻辑都改一遍，漏改了就出bug。

不过话说回来，如果你的权限判断需要动态响应（比如用户角色切换后立即更新UI），`v-if`配合响应式数据会更合适，因为自定义指令的`mounted`只在元素挂载时执行一次，不会自动响应数据变化。各有利弊，按需选择。

## 常见报错解决方案

### 1. v-drag拖拽时选中文字

**现象**：拖拽元素的时候，页面上的文字被选中了，蓝乎乎的一片，拖拽体验极差。

**原因**：浏览器默认的鼠标拖拽行为会选中文字，这是`mousedown`的默认行为。

**解决**：在`mousedown`事件处理函数里调用`e.preventDefault()`阻止默认行为：

```javascript
const onMouseDown = (e) => {
  e.preventDefault(); // 就是这行，不能少
  isDragging = true;
  // ...
};
```

**预防建议**：写拖拽指令的时候，`e.preventDefault()`应该作为`mousedown`的第一行代码，养成习惯就好了。

### 2. v-copy在HTTP环境下不生效

**现象**：本地开发时`v-copy`好好的，部署到HTTP环境后点击复制没反应，控制台报`navigator.clipboard is undefined`。

**原因**：`navigator.clipboard.writeText()`是安全上下文API，只在HTTPS和localhost下可用，HTTP环境下这个API直接不存在。

**解决**：代码里加上降级方案，用`document.execCommand('copy')`兜底（本文的代码已经包含了降级方案）。另外也可以提示用户切换到HTTPS：

```javascript
if (!navigator.clipboard) {
  console.warn("v-copy: 当前环境不支持clipboard API，已使用降级方案");
  // 走execCommand降级逻辑
}
```

**预防建议**：生产环境尽量使用HTTPS。如果必须用HTTP，确保降级方案可用。

### 3. v-lazyload图片闪烁

**现象**：图片加载瞬间会闪一下白光，体验不好。

**原因**：图片从占位符切换到真实图片时，浏览器会先清除旧图片再渲染新图片，中间有个短暂的空白期。

**解决**：初始设置`opacity: 0`，图片加载完成后再过渡到`opacity: 1`（本文代码已实现）：

```javascript
// mounted中
el.style.opacity = "0";
el.style.transition = "opacity 0.3s ease";

// onload中
img.onload = () => {
  img.style.opacity = "1"; // 渐显，不闪
};
```

**预防建议**：还可以给图片设置`min-height`，避免图片加载前后容器高度变化导致的布局抖动。

### 4. v-longpress在移动端触发contextmenu

**现象**：在手机上长按元素，系统的右键菜单（复制/粘贴/选择）先弹出来了，自定义的长按逻辑没执行。

**原因**：移动端长按会触发浏览器的`contextmenu`事件，弹出系统默认菜单。

**解决**：阻止`contextmenu`的默认行为：

```javascript
el.addEventListener("contextmenu", (e) => {
  e.preventDefault();
});
```

**预防建议**：所有涉及长按交互的元素，都应该阻止`contextmenu`默认行为，这是移动端开发的基本操作了。
