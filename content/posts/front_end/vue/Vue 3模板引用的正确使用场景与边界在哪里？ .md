## 一、模板引用的基本概念与核心用途

在Vue 3的声明式渲染模型中，大多数DOM操作都被框架抽象处理，但在某些场景下我们需要直接访问底层DOM元素或子组件实例，这时候就需要用到
**模板引用（Template Refs）**。模板引用通过特殊的`ref`属性实现，它允许我们在组件挂载后获取到具体的DOM节点或组件实例引用，常见用途包括：

- 程序化聚焦输入框
- 初始化第三方库（如Chart.js、地图组件）
- 读取DOM元素的尺寸或位置信息
- 调用子组件的方法（需谨慎使用）

### 1.1 获取DOM元素引用

在Vue 3.5+版本中，推荐使用`useTemplateRef()`辅助函数来获取模板引用：

```vue

<script setup>
  import {useTemplateRef, onMounted} from 'vue'

  // 第一个参数必须与模板中的ref属性值完全匹配
  const inputRef = useTemplateRef('username-input')

  onMounted(() => {
    // 组件挂载后才能访问ref的值，此时可以安全调用DOM方法
    inputRef.value.focus()
  })
</script>

<template>
  <!-- 使用ref属性标记需要引用的元素 -->
  <input ref="username-input" type="text" placeholder="请输入用户名"/>
</template>
```

对于Vue 3.5之前的版本，需要声明一个与模板ref同名的ref变量：

```vue

<script setup>
  import {ref, onMounted} from 'vue'

  // 声明ref变量，名称必须与模板中的ref属性一致
  const inputRef = ref(null)

  onMounted(() => {
    inputRef.value?.focus() // 使用可选链操作符避免null报错
  })
</script>

<template>
  <input ref="inputRef" type="text" placeholder="请输入用户名"/>
</template>
```

### 1.2 获取子组件引用

模板引用也可以用于子组件，此时获取的是子组件的实例对象：

```vue
<!-- Parent.vue -->
<script setup>
  import {useTemplateRef, onMounted} from 'vue'
  import ChildComponent from './ChildComponent.vue'

  const childRef = useTemplateRef('child-component')

  onMounted(() => {
    // 访问子组件暴露的方法或属性
    childRef.value?.sayHello()
  })
</script>

<template>
  <ChildComponent ref="child-component"/>
</template>
```

子组件如果使用了`<script setup>`，需要通过`defineExpose`显式暴露需要被父组件访问的属性或方法：

```vue
<!-- ChildComponent.vue -->
<script setup>
  import {ref} from 'vue'

  const message = ref('Hello from Child!')

  const sayHello = () => {
    console.log(message.value)
  }

  // 显式暴露需要被父组件访问的成员
  defineExpose({
    sayHello,
    message
  })
</script>
```

## 二、避免滥用模板引用的核心原则

模板引用虽然强大，但滥用会导致代码紧耦合、可维护性下降等问题。以下是需要遵循的核心原则：

### 2.1 优先使用Props和Emit进行组件通信

官网明确指出：**组件引用应该只在绝对必要时使用**
，大多数父子组件交互场景应该优先使用Props（数据向下传递）和Emit（事件向上传递）。模板引用会创建父子组件之间的紧耦合，导致子组件的内部实现细节暴露给父组件，增加了代码的维护成本。

**反例（不推荐）**：

```vue
<!-- Parent.vue -->
<script setup>
  import {useTemplateRef, onMounted} from 'vue'
  import Counter from './Counter.vue'

  const counterRef = useTemplateRef('counter')

  const incrementCounter = () => {
    // 直接调用子组件内部方法，紧耦合
    counterRef.value?.count++
  }
</script>

<template>
  <Counter ref="counter"/>
  <button @click="incrementCounter">增加计数</button>
</template>
```

**正例（推荐）**：

```vue
<!-- Parent.vue -->
<script setup>
  import {ref} from 'vue'
  import Counter from './Counter.vue'

  const count = ref(0)

  const incrementCount = () => {
    count.value++
  }
</script>

<template>
  <Counter :count="count"/>
  <button @click="incrementCount">增加计数</button>
</template>

<!-- Counter.vue -->
<script setup>
  defineProps(['count'])
</script>

<template>
  <div>当前计数：{{ count }}</div>
</template>
```

### 2.2 避免在模板中过度使用Ref

Vue的声明式渲染模型旨在让我们专注于数据逻辑而非DOM操作。如果发现自己频繁使用ref来操作DOM，应该思考是否有更符合Vue设计理念的方式：

- 显示/隐藏元素：使用`v-if`或`v-show`而非手动修改DOM的style
- 动态样式：使用`:class`或`:style`绑定而非直接操作classList
- 列表渲染：使用`v-for`而非手动创建DOM元素

### 2.3 注意Ref的生命周期时机

模板引用只能在组件挂载后才能获取到有效值，在组件挂载前（如`setup`函数执行时）访问ref的值会得到`null`。因此，必须在`onMounted`
钩子或`watchEffect`中处理ref的访问逻辑：

```vue

<script setup>
  import {useTemplateRef, watchEffect} from 'vue'

  const inputRef = useTemplateRef('search-input')

  // 使用watchEffect自动处理ref的变化，包括从null到DOM元素的转换
  watchEffect(() => {
    if (inputRef.value) {
      console.log('输入框已挂载，可以操作DOM了')
    } else {
      console.log('输入框未挂载或已被卸载')
    }
  })
</script>
```

## 三、模板引用的调试技巧

### 3.1 处理v-for中的Ref

当在`v-for`中使用ref时，对应的ref变量会自动成为一个数组，包含所有生成的DOM元素或组件实例。需要注意的是，数组的顺序并不一定与源数据完全一致，因此不应该依赖数组索引来定位元素：

```vue

<script setup>
  import {ref, useTemplateRef, onMounted} from 'vue'

  const items = ref(['Item 1', 'Item 2', 'Item 3'])
  const itemRefs = useTemplateRef('list-item')

  onMounted(() => {
    // 输出所有列表项的DOM元素
    console.log(itemRefs.value)
  })
</script>

<template>
  <ul>
    <li v-for="item in items" :key="item" ref="list-item">
      {{ item }}
    </li>
  </ul>
</template>
```

### 3.2 使用函数式Ref

除了字符串形式的ref，还可以使用函数式ref来更灵活地管理引用。函数式ref会在元素挂载和卸载时被调用，参数分别是DOM元素（或组件实例）和
`null`：

```vue

<script setup>
  import {ref} from 'vue'

  const inputElement = ref(null)

  // 函数式ref：元素挂载时保存引用，卸载时清空
  const setInputRef = (el) => {
    inputElement.value = el
  }

  const focusInput = () => {
    inputElement.value?.focus()
  }
</script>

<template>
  <input :ref="setInputRef" type="text"/>
  <button @click="focusInput">聚焦输入框</button>
</template>
```

### 3.3 使用Vue DevTools调试Ref

Vue DevTools提供了直观的ref调试功能：

1. 打开Vue DevTools并切换到"Components"标签
2. 选中目标组件后，在右侧面板的"Refs"部分可以查看所有模板引用的当前值
3. 对于DOM元素引用，可以直接在DevTools中查看和修改其属性

## 四、课后Quiz

### 问题1：在什么场景下应该优先使用模板引用而不是Props/Emit？

**答案解析**：
当需要执行以下操作时，可以考虑使用模板引用：

1. 对DOM元素执行原生API操作（如focus、scrollIntoView）
2. 初始化第三方JavaScript库（如在canvas上初始化图表）
3. 读取DOM元素的尺寸或位置信息（如getBoundingClientRect）
4. 调用子组件中与UI相关的方法（如打开模态框）

但需要注意，即使在这些场景下，也应该尽量将业务逻辑封装在组件内部，通过Props控制状态，仅在必要时使用模板引用执行DOM级操作。

### 问题2：在Vue 3.5版本前后，获取模板引用的方式有什么主要区别？

**答案解析**：

- Vue 3.5及以后：推荐使用`useTemplateRef()`辅助函数，只需传入模板中ref属性的字符串值，无需手动声明ref变量，代码更简洁且类型推断更准确。
- Vue 3.5之前：需要手动声明一个与模板ref属性同名的ref变量，框架会自动将DOM元素或组件实例赋值给该变量。

### 问题3：为什么在使用`<script setup>`的子组件中，父组件无法通过模板引用访问其内部属性？

**答案解析**：
使用`<script setup>`的组件默认是私有的，所有内部属性和方法都不会暴露给父组件。这是为了鼓励组件间通过Props和Emit进行通信，避免紧耦合。如果需要让父组件访问子组件的特定属性或方法，必须使用
`defineExpose`宏显式暴露这些成员。

## 五、常见报错解决方案

### 错误1：Cannot read property 'focus' of null

**错误原因**：在组件未挂载时尝试访问模板引用的值，此时ref的值为null。
**解决方法**：

1. 将操作放在`onMounted`钩子中执行
2. 使用`watchEffect`监听ref的变化，并在值存在时执行操作
3. 使用可选链操作符（`?.`）避免报错

```vue
// 推荐写法
onMounted(() => {
inputRef.value?.focus()
})

// 或者
watchEffect(() => {
if (inputRef.value) {
inputRef.value.focus()
}
})
```

### 错误2：无法访问子组件的内部属性或方法

**错误原因**：

1. 子组件使用了`<script setup>`但未通过`defineExpose`暴露属性
2. 子组件使用了Options API但未通过`expose`选项限制访问范围
   **解决方法**：

- 对于`<script setup>`组件：使用`defineExpose`暴露需要被访问的成员
- 对于Options API组件：使用`expose`选项指定可访问的属性和方法

```vue
<!-- 子组件示例 -->
<script setup>
  const internalData = ref('私有数据')
  const publicMethod = () => {
    console.log('这是公共方法')
  }

  // 仅暴露需要被父组件访问的成员
  defineExpose({
    publicMethod
  })
</script>
```

### 错误3：v-for中的ref数组顺序与源数据不一致

**错误原因**：Vue的虚拟DOM更新策略可能会导致ref数组的顺序与源数据不完全匹配，尤其是在使用复杂的列表更新逻辑时。
**解决方法**：

1. 不要依赖ref数组的顺序来定位元素
2. 如果需要精准定位，为每个元素添加唯一的`key`属性，并通过`key`来关联数据和DOM元素
3. 使用函数式ref将元素引用与数据项关联起来
