## 一、Props传值：父组件到子组件的桥梁

### 1.1 什么是Props

Props（Properties的缩写）是Vue组件间通信的基础方式之一，用于实现**父组件向子组件传递数据**。可以把Props想象成组件的"
输入参数"，子组件通过定义Props来接收父组件传递过来的数据，就像函数接收参数一样。

### 1.2 Props的声明方式

在Vue 3中，我们有多种方式声明Props，最常用的是在`<script setup>`中使用`defineProps()`宏：

#### 1.2.1 数组式声明（基础版）

```vue

<script setup>
  // 声明一个名为"message"的Props
  const props = defineProps(['message'])

  // 在组件中使用Props
  console.log(props.message)
</script>

<template>
  <div>{{ message }}</div>
</template>
```

#### 1.2.2 对象式声明（带类型验证）

```vue

<script setup>
  // 声明Props并指定类型
  const props = defineProps({
    title: String, // 字符串类型
    likes: Number, // 数字类型
    isPublished: Boolean, // 布尔类型
    commentIds: Array, // 数组类型
    author: Object // 对象类型
  })
</script>
```

#### 1.2.3 TypeScript类型声明（进阶版）

如果使用TypeScript，可以直接用类型注解声明Props：

```vue

<script setup lang="ts">
  // 使用TypeScript类型声明Props
  defineProps<{
    title?: string
    likes: number
  }>()
</script>
```

### 1.3 Props的传递细节

#### 1.3.1 大小写规范

- 子组件中使用**驼峰式（camelCase）**声明Props，比如`greetingMessage`
- 父组件中使用**短横线分隔式（kebab-case）**传递Props，比如`greeting-message`

```vue
<!-- 父组件 -->
<ChildComponent greeting-message="Hello Vue 3!"/>

<!-- 子组件 -->
<script setup>
  const props = defineProps(['greetingMessage'])
</script>
<template>
  <div>{{ greetingMessage }}</div>
</template>
```

#### 1.3.2 静态与动态传递

- 静态传递：直接传递固定值

```vue

<ChildComponent title="My Blog Post"/>
```

- 动态传递：使用`v-bind`（或简写`:`）传递变量或表达式

```vue

<ChildComponent :title="post.title" :likes="post.likes"/>
<ChildComponent :title="post.title + ' by ' + post.author"/>
```

#### 1.3.3 传递不同类型的值

```vue
<!-- 数字类型 -->
<ChildComponent :likes="42"/>

<!-- 布尔类型 -->
<ChildComponent is-published/> <!-- 等价于:is-published="true" -->
<ChildComponent :is-published="false"/>

<!-- 数组类型 -->
<ChildComponent :comment-ids="[234, 266, 273]"/>

<!-- 对象类型 -->
<ChildComponent :author="{ name: 'John', company: 'Vue.js' }"/>
```

### 1.4 单向数据流原则

Vue遵循**单向数据流**原则：父组件的Props更新会自动传递给子组件，但子组件不能直接修改Props。如果尝试修改，Vue会在控制台发出警告。

如果需要修改Props的值，有两种常见方案：

1. **复制为本地数据**：当Props作为初始值使用时

```vue

<script setup>
  const props = defineProps(['initialCount'])
  // 将Props复制为本地响应式数据
  const count = ref(props.initialCount)
</script>
```

2. **使用计算属性**：当需要对Props进行转换时

```vue

<script setup>
  const props = defineProps(['size'])
  // 使用计算属性处理Props
  const normalizedSize = computed(() => props.size.trim().toLowerCase())
</script>
```

### 1.5 Props验证

为了提高组件的健壮性，我们可以对Props进行验证，确保父组件传递的数据符合预期。

```vue

<script setup>
  const props = defineProps({
    // 基础类型检查
    id: Number,

    // 多种可能类型
    title: [String, Number],

    // 必填的字符串
    content: {
      type: String,
      required: true
    },

    // 带默认值的数字
    likes: {
      type: Number,
      default: 0
    },

    // 带默认值的对象
    author: {
      type: Object,
      // 对象或数组的默认值必须从工厂函数返回
      default() {
        return {name: 'Anonymous'}
      }
    },

    // 自定义验证函数
    status: {
      validator(value) {
        // 验证值是否为指定的字符串之一
        return ['draft', 'published', 'archived'].includes(value)
      }
    }
  })
</script>
```

### 1.6 Props传递流程图

```
父组件
    |
    v
定义要传递的数据
    |
    v
在模板中通过属性传递给子组件
    |
    v
子组件使用defineProps声明接收
    |
    v
子组件模板或脚本中使用Props数据
```

## 二、自定义事件（$emit）：子组件到父组件的通信

### 2.1 什么是自定义事件

自定义事件是Vue实现**子组件向父组件传递数据**的主要方式。子组件通过`$emit`方法触发一个自定义事件，父组件通过`v-on`（或简写
`@`）监听这个事件并处理数据。

### 2.2 触发和监听事件

#### 2.2.1 子组件触发事件

```vue
<!-- 子组件 -->
<template>
  <button @click="$emit('increase')">点击增加</button>
</template>

<script setup>
  // 也可以在脚本中触发事件
  const emit = defineEmits(['increase'])

  function handleClick() {
    emit('increase')
  }
</script>
```

#### 2.2.2 父组件监听事件

```vue
<!-- 父组件 -->
<template>
  <ChildComponent @increase="count++"/>
  <p>当前计数：{{ count }}</p>
</template>

<script setup>
  import {ref} from 'vue'

  const count = ref(0)
</script>
```

### 2.3 传递事件参数

有时候我们需要在触发事件时传递额外的数据，比如点击按钮时传递具体的增加值：

```vue
<!-- 子组件 -->
<template>
  <button @click="$emit('increaseBy', 2)">点击增加2</button>
</template>

<script setup>
  const emit = defineEmits(['increaseBy'])
</script>
```

```vue
<!-- 父组件 -->
<template>
  <ChildComponent @increase-by="(n) => count += n"/>
  <!-- 或者使用方法处理 -->
  <ChildComponent @increase-by="handleIncrease"/>
</template>

<script setup>
  import {ref} from 'vue'

  const count = ref(0)

  function handleIncrease(n) {
    count.value += n
  }
</script>
```

### 2.4 声明触发的事件

为了更好地文档化组件，我们可以显式声明组件会触发的事件：

```vue

<script setup>
  // 数组式声明
  const emit = defineEmits(['increase', 'increaseBy'])

  // 对象式声明（支持验证）
  const emit = defineEmits({
    increase: null, // 无验证
    increaseBy: (value) => {
      // 验证参数是否为数字
      return typeof value === 'number'
    }
  })
</script>
```

### 2.5 自定义事件流程图

```
子组件
    |
    v
处理用户交互或业务逻辑
    |
    v
使用$emit触发自定义事件（可携带参数）
    |
    v
父组件通过v-on监听事件
    |
    v
父组件执行事件处理函数，接收并处理数据
```

## 三、课后Quiz

### 问题1：如何在Vue 3中声明一个接收字符串类型的Props？

**答案**：有三种常见方式：

1. 数组式声明：

```vue

<script setup>
  defineProps(['message'])
</script>
```

2. 对象式声明：

```vue

<script setup>
  defineProps({
    message: String
  })
</script>
```

3. TypeScript类型声明：

```vue

<script setup lang="ts">
  defineProps<{
    message: string
  }>()
</script>
```

### 问题2：子组件如何向父组件传递数据？

**答案**：子组件通过`$emit`方法触发自定义事件，父组件通过`v-on`监听事件并接收数据：

```vue
// 子组件
const emit = defineEmits(['sendData'])
emit('sendData', { name: 'Vue 3' })

// 父组件
<ChildComponent @send-data="handleData"/>
function handleData(data) {
console.log(data) // { name: 'Vue 3' }
}
```

### 问题3：什么是单向数据流？为什么要遵循这个原则？

**答案**：单向数据流指的是Props的数据只能从父组件流向子组件，子组件不能直接修改Props。遵循这个原则的原因是：

1. 保持数据流向清晰，便于理解和调试
2. 防止子组件意外修改父组件的状态，导致应用状态混乱
3. 提高组件的可维护性和复用性

## 四、常见报错解决方案

### 报错1：[Vue warn]: Invalid prop: type check failed for prop "xxx". Expected String, got Number

**原因**：父组件传递的Props类型与子组件声明的类型不匹配。
**解决办法**：

1. 检查父组件传递的数据类型，确保与子组件声明的类型一致
2. 如果需要接受多种类型，可以在子组件中使用数组式声明，比如`defineProps({ xxx: [String, Number] })`

### 报错2：[Vue warn]: Avoid mutating a prop directly since the value will be overwritten whenever the parent component re-renders.

**原因**：子组件直接修改了Props的值，违反了单向数据流原则。
**解决办法**：

1. 如果需要将Props作为初始值，复制为子组件的本地响应式数据
2. 如果需要修改数据，通过自定义事件通知父组件修改原始数据
3. 如果需要对Props进行转换，使用计算属性

### 报错3：[Vue warn]: Extraneous non-emitted event(s): "xxx".

**原因**：父组件监听了子组件未声明的事件。
**解决办法**：

1. 在子组件中使用`defineEmits`声明所有会触发的事件
2. 检查父组件的事件监听名称是否正确
