## 一、指令用在普通元素上，一切正常

前面几篇文章咱们聊了不少自定义指令的东西——什么v-focus让输入框自动聚焦、v-color给元素上色、v-permission控制按钮权限，用起来那叫一个丝滑。

但你有没有注意到，这些指令都是直接贴在原生HTML元素上的？比如：

```html
<!-- v-focus直接贴在input上，没毛病 -->
<input v-focus placeholder="我会自动获取焦点" />

<!-- v-color直接贴在div上，也没毛病 -->
<div v-color="'red'">我是红色的文字</div>
```

为啥这些用起来都好好的？原因很简单——指令从诞生那天起，就是冲着DOM元素去的。它拿到的第一个参数`el`就是那个真实的DOM节点，你想操作啥就操作啥，清清楚楚明明白白。

```js
const vFocus = {
  mounted(el) {
    // el就是那个input元素，直接调用focus()方法
    el.focus();
  },
};
```

这就好比你给一把锁配了把钥匙，锁就在那里，钥匙插进去就能开。但是，当你把指令贴到组件上的时候，事情就开始变得微妙了……

## 二、指令贴到组件上会发生啥？

### 单根组件：指令偷偷溜到根节点上

当你把自定义指令贴到一个组件上时，Vue会做一个"自动搬运"的操作——把指令应用到组件模板的根节点上。这个行为和透传attributes（Fallthrough Attributes）非常像。

来看个具体例子：

```html
<!-- 父组件中使用 -->
<MyComponent v-demo="test" />
```

```html
<!-- MyComponent的模板 -->
<template>
  <div>
    <!-- v-demo指令会被应用在这个div上 -->
    <span>My component content</span>
  </div>
</template>
```

你写在`<MyComponent>`上的`v-demo`指令，实际上被Vue搬到了组件内部的`<div>`上。说白了，指令根本不知道你组件里面长啥样，它就认准了最外面那层壳。

咱们用一个完整的例子来验证一下：

```vue
<!-- App.vue -->
<script setup>
import { ref } from "vue";
import MyInput from "./MyInput.vue";

const msg = ref("hello");

// 定义一个v-log指令，用来打印它绑定的元素信息
const vLog = {
  mounted(el) {
    console.log("v-log绑定的元素标签名：", el.tagName);
    console.log("v-log绑定的元素内容：", el.textContent);
  },
};
</script>

<template>
  <!-- 把v-log贴到组件上 -->
  <MyInput v-log="msg" />
</template>
```

```vue
<!-- MyInput.vue -->
<script setup>
defineProps(["modelValue"]);
defineEmits(["update:modelValue"]);
</script>

<template>
  <!-- 这是根节点，v-log会贴到这个div上 -->
  <div class="input-wrapper">
    <input
      :value="modelValue"
      @input="$emit('update:modelValue', $event.target.value)"
    />
  </div>
</template>
```

打开控制台，你会看到打印出来的是：

```
v-log绑定的元素标签名： DIV
v-log绑定的元素内容： [input里的内容]
```

看到了吧？指令操作的是那个`<div class="input-wrapper">`，而不是里面的`<input>`。如果你的本意是让指令操作`<input>`元素，那不好意思，它跑偏了。

用流程图来理解这个过程：

```
┌─────────────────────────────────────────────┐
│           父组件模板                          │
│                                             │
│   <MyComponent v-demo="test" />             │
│         │                                   │
│         │  Vue自动将指令搬运到                 │
│         │  组件的根节点                        │
│         ▼                                   │
├─────────────────────────────────────────────┤
│           MyComponent 模板                   │
│                                             │
│   <div v-demo="test">  ◄── 指令落在这里       │
│     <span>content</span>                    │
│   </div>                                    │
└─────────────────────────────────────────────┘
```

## 三、多根组件：指令直接摆烂

Vue 3有一个很酷的特性——组件可以有多个根节点，这叫Fragment。比如：

```html
<!-- MyComponent的模板 -->
<template>
  <p>段落1</p>
  <p>段落2</p>
</template>
```

这在Vue 2里是不行的，Vue 2必须只有一个根元素。但Vue 3说：没事，多几个根也挺好。

然而，当你把指令贴到这种多根组件上的时候，指令就懵了——到底该贴到哪个根节点上？第一个`<p>`？第二个`<p>`？还是都贴？

Vue给出的答案是：**都不贴，直接忽略，然后给你甩一个警告**。

```html
<!-- 父组件 -->
<MultiRootComponent v-demo="test" />
```

```vue
<!-- MultiRootComponent.vue -->
<template>
  <p>我是第一个段落</p>
  <p>我是第二个段落</p>
</template>
```

控制台会输出类似这样的警告：

```
[Vue warn]: Runtime directive used on component with non-element root node. The directives will not function as intended.
```

翻译过来就是：你把指令用在了多根节点的组件上，指令没法正常工作。

这就像你让快递员把包裹送到一栋楼，结果这栋楼有多个入口，快递员不知道该放哪个门口，干脆就不送了，还给你发了条消息说"你自己来拿吧"。

```
┌─────────────────────────────────────────────┐
│           父组件模板                          │
│                                             │
│   <MultiRootComponent v-demo="test" />      │
│         │                                   │
│         │  Vue发现组件有多个根节点              │
│         │  不知道该贴哪个                      │
│         │                                   │
│         ▼                                   │
│    ⚠️ 警告！指令被忽略                        │
├─────────────────────────────────────────────┤
│       MultiRootComponent 模板                │
│                                             │
│   <p>段落1</p>   ◄── 贴这个？                │
│   <p>段落2</p>   ◄── 还是这个？               │
│                                             │
│   结果：哪个都不贴，直接摆烂                    │
└─────────────────────────────────────────────┘
```

## 四、指令和透传attributes的异同

说到这里，你可能想到了一个东西——透传attributes（Fallthrough Attributes）。它和指令在组件上的行为确实很像，但又不完全一样。咱们来掰扯掰扯。

### 相同点

不管是透传attributes还是自定义指令，在单根组件上都会自动应用到根节点。比如：

```html
<!-- 父组件 -->
<MyComponent class="big" v-demo="test" />
```

```vue
<!-- MyComponent.vue（单根） -->
<template>
  <!-- class和v-demo都会自动来到这个div上 -->
  <div>hello</div>
</template>
```

### 不同点

关键区别在于多根组件的处理方式和手动控制能力。

**透传attributes在多根组件中的处理**：你可以通过`v-bind="$attrs"`手动指定attributes要透传到哪个元素上。

```vue
<!-- MultiRootComponent.vue -->
<template>
  <p v-bind="$attrs">段落1 - attributes会到我这里</p>
  <p>段落2</p>
</template>
```

**自定义指令在多根组件中的处理**：完全不支持手动指定。你不能写什么`v-demo="$attrs"`之类的，指令根本没有这种机制。

```vue
<!-- 这是不可能的！指令没有类似v-bind="$attrs"的写法 -->
<template>
  <p v-???="???">段落1</p>
  <p>段落2</p>
</template>
```

用表格来对比更清晰：

| 特性                                | 透传attributes            | 自定义指令           |
| ----------------------------------- | ------------------------- | -------------------- |
| 单根组件                            | 自动透传到根节点          | 自动应用到根节点     |
| 多根组件                            | 需要手动`v-bind="$attrs"` | 直接忽略+警告        |
| 手动指定目标元素                    | 可以，用`v-bind="$attrs"` | 不可以，没有这种机制 |
| 是否可通过`inheritAttrs: false`控制 | 可以                      | 不可以               |
| 本质                                | HTML属性，数据驱动        | DOM操作，行为驱动    |

所以你看，attributes好歹还有个"后门"让你手动控制，指令连后门都没有。这也是为啥官方不太推荐在组件上用指令的原因之一。

## 五、官方态度：不推荐在组件上用自定义指令

Vue官方文档在"在组件上使用"这一节明确标注了"不推荐"：

> 不推荐在组件上使用自定义指令。当组件具有多个根节点时可能会出现预期外的行为。

那为啥不推荐呢？原因有这么几个：

### 组件是封装的，内部DOM结构可能变化

组件的本质就是封装。你用组件的时候，不需要也不应该关心它内部的DOM长什么样。但指令恰恰就是要操作DOM的，这就产生了矛盾。

```vue
<!-- 今天组件的模板是这样的 -->
<template>
  <div>
    <input />
  </div>
</template>

<!-- 明天你重构了一下，模板变成了这样 -->
<template>
  <label>
    <span>请输入：</span>
    <input />
  </label>
</template>
```

根节点从`<div>`变成了`<label>`，你贴在组件上的指令操作的对象也跟着变了。如果指令的逻辑对元素类型有要求（比如要求是`<input>`），那就直接翻车了。

### 指令依赖具体的DOM元素，耦合太强

指令和DOM是强绑定的关系。你在组件上贴指令，就等于在说"我要操作你这个组件内部的某个DOM元素"。但组件的内部实现是你管不着的，人家随时可能改。这种强耦合在软件工程里是大忌。

### 那该用啥替代？

Vue官方推荐的做法是：

1. **用组合式函数（Composables）代替**——这是最推荐的方案。组合式函数操作的是响应式数据，不直接碰DOM，天然解耦。
2. **在组件内部使用指令**——如果确实需要指令，就在组件自己的模板里用，别从外面贴。
3. **通过props/emits和组件通信**——让组件自己决定怎么处理，而不是从外面强行操作它的DOM。

## 六、如果非要在组件上用指令怎么办？

虽然官方不推荐，但有时候你可能就是有这个需求。那咱们就来看看几种变通方案。

### 方案一：确保组件是单根节点

最简单的办法——保证你的组件只有一个根元素。这样指令至少能正常工作，不会出现被忽略的情况。

```vue
<!-- SingleRootComponent.vue -->
<script setup>
defineProps(["value"]);
</script>

<template>
  <!-- 只有一个根元素，指令会贴到这个div上 -->
  <div class="container">
    <input :value="value" />
  </div>
</template>
```

```vue
<!-- App.vue -->
<script setup>
import { ref } from "vue";
import SingleRootComponent from "./SingleRootComponent.vue";

const text = ref("");

const vFocus = {
  mounted(el) {
    // 注意：el是div，不是input！
    // 如果你想聚焦input，需要el.querySelector('input')?.focus()
    const input = el.querySelector("input");
    if (input) input.focus();
  },
};
</script>

<template>
  <SingleRootComponent v-focus :value="text" />
</template>
```

但要注意，即使指令生效了，它操作的也是根节点（`<div>`），不是你想要的`<input>`。如果你需要操作特定的子元素，得在指令里手动查找。

### 方案二：在组件内部使用指令，通过props控制

与其从外面贴指令，不如让组件自己内部用指令，然后通过props来控制行为。

```vue
<!-- SmartInput.vue -->
<script setup>
const props = defineProps({
  modelValue: String,
  autofocus: Boolean, // 通过props控制是否自动聚焦
});

defineEmits(["update:modelValue"]);

// 指令定义在组件内部
const vFocus = {
  mounted(el) {
    el.focus(); // 这里el直接就是input，精准！
  },
};
</script>

<template>
  <!-- 在组件内部使用指令，配合v-if按需启用 -->
  <input
    v-if="autofocus"
    v-focus
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
  />
  <input
    v-else
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>
```

```vue
<!-- App.vue -->
<script setup>
import { ref } from "vue";
import SmartInput from "./SmartInput.vue";

const text = ref("");
</script>

<template>
  <!-- 通过props控制，清晰明了 -->
  <SmartInput v-model="text" autofocus />
  <SmartInput v-model="text" />
</template>
```

这样做的好处是：指令直接贴在`<input>`上，操作的就是`<input>`，不会跑偏。而且组件的使用者只需要传一个prop，不用关心内部实现。

### 方案三：用组合式函数替代指令逻辑

这是最推荐的方案。把指令的逻辑提取成组合式函数，在组件内部调用。

```js
// composables/useAutoFocus.js
import { onMounted, nextTick } from "vue";

export function useAutoFocus(inputRef) {
  onMounted(async () => {
    // 等待DOM渲染完成
    await nextTick();
    if (inputRef.value) {
      inputRef.value.focus();
    }
  });
}
```

```vue
<!-- AutoFocusInput.vue -->
<script setup>
import { ref } from "vue";
import { useAutoFocus } from "../composables/useAutoFocus";

const props = defineProps({
  modelValue: String,
  autofocus: Boolean,
});

defineEmits(["update:modelValue"]);

const inputRef = ref(null);

// 如果需要自动聚焦，调用组合式函数
if (props.autofocus) {
  useAutoFocus(inputRef);
}
</script>

<template>
  <input
    ref="inputRef"
    :value="modelValue"
    @input="$emit('update:modelValue', $event.target.value)"
  />
</template>
```

```vue
<!-- App.vue -->
<script setup>
import { ref } from "vue";
import AutoFocusInput from "./AutoFocusInput.vue";

const text = ref("");
</script>

<template>
  <AutoFocusInput v-model="text" autofocus />
</template>
```

组合式函数的优势很明显：

- 不依赖DOM结构，只依赖ref引用
- 可以在任何组件里复用
- 逻辑清晰，方便测试
- 不会因为组件根节点变化而失效

三种方案的对比：

| 方案                 | 优点             | 缺点                                 |
| -------------------- | ---------------- | ------------------------------------ |
| 确保单根节点         | 简单，改动小     | 指令操作的是根节点，可能不是目标元素 |
| 组件内部用指令+props | 精准控制目标元素 | 需要修改组件内部代码                 |
| 组合式函数替代       | 最灵活，解耦最好 | 需要重构逻辑，改动较大               |

## 课后Quiz

### 问题1：自定义指令应用在多根组件上会怎样？

**答案**：指令会被忽略，同时控制台会抛出一个警告。因为Vue不知道该把指令应用到哪个根节点上，所以干脆就不应用了。这和透传attributes不同——attributes可以通过`v-bind="$attrs"`手动指定目标元素，但指令没有这种机制。

### 问题2：为什么指令不能像attributes那样通过v-bind="$attrs"传递？

**答案**：因为attributes和指令的本质不同。attributes是数据（比如class、style、id），它们可以通过JavaScript对象的形式在模板中传递。而指令是行为（包含mounted、updated等钩子函数），它不是简单的键值对数据，没法塞进`$attrs`对象里。`$attrs`只包含attributes和事件监听器，不包含指令。这是Vue框架层面的设计限制，不是某个bug。

## 常见报错解决方案

### 1. 指令在组件上不生效

**现象**：你把v-xxx贴到组件上了，但指令的钩子函数根本没执行。

**排查步骤**：

第一步，检查组件是不是多根节点。打开组件的模板，看看是不是有多个并列的根元素：

```vue
<!-- 这种就是多根组件，指令不会生效 -->
<template>
  <h1>标题</h1>
  <p>内容</p>
</template>
```

第二步，如果是多根组件，要么改成单根，要么把指令挪到组件内部使用。

第三步，如果确实是单根组件但指令还是不生效，检查指令是否正确注册。在`<script setup>`中，指令变量名必须以`v`开头并使用驼峰命名，比如`vFocus`对应`v-focus`。

### 2. 指令操作了错误的元素

**现象**：指令生效了，但操作的不是你想要的元素。比如你想让input聚焦，结果聚焦的是外层的div。

**原因**：在组件上使用指令时，指令操作的是组件的根节点，不是你想要的那个子元素。

**解决方案**：在指令内部手动查找目标元素：

```js
const vFocus = {
  mounted(el) {
    // el是根节点（比如div），需要找到里面的input
    const input = el.querySelector("input");
    if (input) {
      input.focus();
    } else {
      console.warn("v-focus：找不到input元素");
    }
  },
};
```

或者更好的做法是改用方案二或方案三，在组件内部使用指令或组合式函数。

### 3. 组件重构后指令突然失效

**现象**：之前好好的指令，组件改了一版模板之后就不工作了。

**原因**：组件的根节点类型变了。比如原来根节点是`<div>`，重构后变成了`<template>`包裹的多个元素（变成了多根组件），或者根节点从`<input>`变成了`<div>`。

**解决方案**：

- 重构组件时注意保持根节点结构稳定
- 如果必须改根节点，同步检查所有外部使用的指令
- 最好的办法是避免在组件外部使用指令，改用组合式函数或props控制

**预防建议**：在组件文档或注释中标注"本组件根节点为xxx，外部使用指令将作用于该节点"，方便后续维护。
