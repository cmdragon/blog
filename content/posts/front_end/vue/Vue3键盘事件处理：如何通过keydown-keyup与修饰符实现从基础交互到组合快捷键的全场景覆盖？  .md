### 一、监听键盘事件：keydown 与 keyup 的核心区别

在Web开发中，键盘交互是提升用户体验的关键——比如登录页按回车提交、搜索框实时联想、文本编辑器快捷键。Vue3将键盘事件简化为两个核心类型：
`keydown`（按键**按下**时触发）和`keyup`（按键**松开**时触发），两者的差异直接影响交互逻辑的设计：

| 事件类型    | 触发时机         | 应用场景          |  
|---------|--------------|---------------|  
| keydown | 按下瞬间，长按会连续触发 | 实时搜索、连续删除字符   |  
| keyup   | 松开时触发，仅一次    | 按回车提交表单、单次快捷键 |  

举个**实时搜索**的实际案例：用户输入字符时立刻更新搜索结果，此时`keydown`能精准捕捉每一次按键输入；而如果是“输入完成后按回车提交”，则用
`keyup.enter`更合理——避免用户未输完就触发请求。

```vue
<!-- 实时搜索框示例 -->
<template>
  <input
      type="text"
      v-model="keyword"
      @keydown="handleRealTimeSearch"
      placeholder="输入关键词，实时联想"
  />
</template>

<script setup>
  import {ref} from 'vue'

  const keyword = ref('')

  // 实时搜索逻辑（实际项目需加防抖）
  const handleRealTimeSearch = () => {
    console.log('正在搜索：', keyword.value)
    // 调用接口：axios.get('/api/search', { params: { keyword: keyword.value } })
  }
</script>
```

### 二、常用按键修饰符：告别繁琐的 keyCode 判断

早期JavaScript处理键盘事件需要手动判断`event.keyCode`（比如`keyCode === 13`代表回车），但`keyCode`已被浏览器废弃。Vue3贴心地为
**常用按键**提供了**修饰符**，直接通过`.`连接事件和按键名称，代码更简洁。

#### 1. 基础按键修饰符

Vue3支持的常用单键修饰符：

- 确认类：`.enter`（回车）、`.tab`（制表）
- 删除类：`.delete`（删除/退格）
- 控制类：`.esc`（退出）、`.space`（空格）
- 方向类：`.up`（上）、`.down`（下）、`.left`（左）、`.right`（右）

**实际场景**：登录表单按回车提交。

```vue

<template>
  <form @submit.prevent="login">
    <input v-model="username" placeholder="用户名" @keyup.enter="login"/>
    <input type="password" v-model="password" placeholder="密码" @keyup.enter="login"/>
    <button type="submit">登录</button>
  </form>
</template>

<script setup>
  import {ref} from 'vue'

  const username = ref('')
  const password = ref('')

  const login = () => {
    if (!username.value || !password.value) return alert('请填写完整信息')
    console.log('登录请求：', username.value, password.value)
  }
</script>
```  

解释：输入框绑定`@keyup.enter`，用户松开回车键时触发`login`，无需手动判断`keyCode`。

### 三、自定义按键修饰符：处理特殊快捷键

Vue3的内置修饰符覆盖了常用场景，但如果需要**特殊单键**（如F1~F12、CapsLock）或**组合快捷键**
（如Ctrl+S、Shift+Enter），可以通过以下方式实现：

#### 1. 特殊单键：用按键名称

Vue3支持直接使用按键的**名称**（如'F1'、'CapsLock'）作为修饰符。例如：

- 按F1显示帮助：`@keydown.f1="showHelp"`
- 按CapsLock提示大小写：`@keydown.capslock="toggleCapsHint"`

#### 2. 组合快捷键：修饰键+普通键

组合快捷键需要结合**修饰键**（`.ctrl`/`.alt`/`.shift`/`.meta`）和普通键修饰符。例如：

- Ctrl+S保存：`@keydown.ctrl.s="save"`
- Shift+Enter换行：`@keydown.shift.enter="insertLineBreak"`
- Mac下Command+D复制：`@keydown.meta.d="copy"`

**代码示例：文本编辑器快捷键**

```vue

<template>
  <textarea
      v-model="content"
      @keydown.ctrl.s="saveContent"
      @keydown.shift.enter="insertLine"
      rows="5"
      placeholder="按Ctrl+S保存，Shift+Enter换行"
  ></textarea>
</template>

<script setup>
  import {ref} from 'vue'

  const content = ref('')

  const saveContent = (e) => {
    e.preventDefault() // 阻止浏览器默认的Ctrl+S行为（如弹出保存对话框）
    console.log('保存内容：', content.value)
  }

  const insertLine = (e) => {
    e.preventDefault() // 阻止默认的Enter换行
    content.value += '\n' // 手动插入换行符
  }
</script>
```  

解释：

- `@keydown.ctrl.s`：按下Ctrl+S时触发`saveContent`，并通过`e.preventDefault()`阻止浏览器默认行为。
- `@keydown.shift.enter`：按下Shift+Enter时触发`insertLine`，手动插入换行符。

### 四、课后 Quiz：巩固知识点

#### 问题1：如何用Vue3实现“按F5键时阻止页面刷新”？

**答案**：使用`@keydown.f5.prevent`，其中`.prevent`修饰符阻止浏览器默认的F5刷新行为。

```vue

<template>
  <div @keydown.f5.prevent="handlePreventRefresh">
    按F5不会刷新页面
  </div>
</template>

<script setup>
  const handlePreventRefresh = () => {
    alert('已阻止F5刷新，请使用按钮刷新')
  }
</script>
```  

#### 问题2：如何实现“按Alt+Delete永久删除文件”？

**答案**：使用组合修饰符`@keydown.alt.delete`。

```vue

<template>
  <button @keydown.alt.delete="permanentDelete">
    按Alt+Delete永久删除
  </button>
</template>

<script setup>
  const permanentDelete = () => {
    console.log('永久删除文件')
  }
</script>
```

### 五、常见报错与解决方法

#### 1. 按键修饰符不生效

**原因**：

- 按键名称拼写错误（如把'f1'写成'fone'）；
- 使用了已废弃的`keyCode`（如`@keydown.13`，Vue3不再支持）。

**解决**：

- 通过`console.log(event.key)`在事件处理函数中查看按键的正确名称；
- 将`keyCode`替换为按键名称（如`@keydown.13`→`@keydown.enter`）。

#### 2. 组合快捷键不触发

**原因**：

- 未同时按下组合键（如Ctrl+S需要同时按住Ctrl和S）；
- 浏览器默认行为阻止了事件（如Ctrl+W会关闭标签页）。

**解决**：

- 确保同时按下组合键；
- 在事件处理函数中添加`event.preventDefault()`阻止默认行为。

#### 3. 阻止默认行为无效

**原因**：

- 事件类型错误（`keyup`事件的默认行为可能已执行，应改用`keydown`）；
- `.prevent`修饰符位置错误（如`@keydown.s.ctrl.prevent`，应改为`@keydown.prevent.ctrl.s`）。

**解决**：

- 改用`keydown`事件（如`@keydown.prevent.ctrl.s`）；
- 确保`.prevent`修饰符在事件类型后。
